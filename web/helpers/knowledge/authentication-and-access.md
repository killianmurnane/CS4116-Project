Users log in with email and password. Passwords are verified against a stored hash. If credentials are valid the system stores session values including user id email role and name. Admin users are redirected to the admin page after login while standard users go to the dashboard. Banned users cannot log in.

Registration requires email first name last name gender date of birth location password and password confirmation. The email must be valid and not already exist. Gender must be male female or other. Password must be at least 8 characters and the confirmation must match. Users must be at least 18 years old. Successful registration creates both a user account and a profile row.

Logged-out users trying to access protected pages are redirected to the login page. Only admins can access the admin page. Logging out clears the session destroys the session cookie and redirects to the login page.
